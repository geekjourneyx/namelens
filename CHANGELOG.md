# Changelog

All notable changes to NameLens are documented in this file.

The format follows Keep a Changelog, and this project uses semantic versioning.

## [1.0.0] - 2026-06-23

### Added

- Initial `namelens` Agent Skill for researching, stress-testing, and choosing names for open-source projects.
- Quick, Standard, and Deep naming research modes.
- Naming workflow covering brief extraction, namespace prioritization, semantic territories, candidate generation, collision research, ordinal evaluation, adversarial review, and launch identity design.
- Explicit operating checkpoints for scope escalation, critical conflicts, clearance language, and missing live access.
- Failure handling table for unavailable research sources, language uncertainty, blocked namespaces, same-category collisions, partial screens, and legal-clearance requests.
- Do-not-do blacklist for unsafe naming claims, fake precision, false etymology, hidden collisions, SEO keyword stuffing, and red-candidate polishing.
- Reference documents for naming patterns, collision research, and Standard/Deep output templates.
- Test prompts and eval cases for CLI, package, consumer app, and project-name comparison scenarios.
- README, MIT license, and README banner assets.
