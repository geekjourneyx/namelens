# Naming Report Template

Use this structure for Standard and Deep outputs. Compress it for Quick mode.

## 1. Decision

State the winner immediately.

> Choose **[Name]**. It best balances [top strengths]. Its main remaining risk is [risk]. Keep **[Backup]** reserved until [critical checks] are complete.

## 3. Search scope

Include:

- date researched
- web search languages
- GitHub and registries checked
- app stores, domains, handles, or trademark registers checked
- material checks that remain unverified

## 4. Naming territories

Summarize the independent lanes explored:

| Lane | Idea | Example semantic field |
|---|---|---|
| Clear compound | immediate relationship | source + path |
| Suggestive | result or feeling | clarity, signal |
| Coined | pronounceable distinctive form | familiar phonetic fragments |
| Cross-language | verified root | trace, light, route |
| Metaphor | concrete behavior | beacon, loom, relay |
| Technical | terse command identity | probe, index, route |

## 5. Candidate matrix

| Name | Origin / construction | Pronunciation | Semantic fit | Transmission | Search ownership | Critical namespace | Collision | Difficulty | Main risk |
|---|---|---|---|---|---|---|---|---|---|
| Candidate | verified or root-inspired | simple guide | Strong | Strong | Medium | Clear | Yellow | Medium | unrelated entity |

Use Strong / Medium / Weak and Clear / Workaround / Blocked / Unverified rather than fake decimal precision.

## 7. Rejected or dangerous candidates

List candidates that looked attractive but failed, and explain the failure:

- generic or impossible-to-own query

### Entity definition

Write one stable factual sentence:

> [Name] is a [category] for [audience] that [core outcome].

Use the same definition in README introductions, package metadata, documentation, website copy, and launch materials.

### Category descriptor

Prefer:

```text
[category] for [audience or outcome]
```

Avoid stuffing every feature or provider into the brand itself.

## 9. Evidence log

| Candidate | Check | Finding | Relevance | Confidence | Date |
|---|---|---|---|---|---|
| Name | GitHub exact search | active unrelated repository | low | High | YYYY-MM-DD |

Include citations or direct references in the report.

## 10. Reservation checklist

Prioritize only the surfaces that matter:

- reserve GitHub repository or organization
- reserve package/module name
- confirm CLI command does not conflict with common system tools
- register domain and social handles if launch-critical
- create canonical spelling and capitalization rules
- add repository About text and topics
- add stable entity definition to README
- conduct professional trademark review if commercial stakes warrant it

## 11. Final wording

Use:

> In the searches performed on YYYY-MM-DD, no obvious active same-category exact collision was found.

Avoid:

> The name is available, guaranteed, trademark-safe, or SEO-proof.
